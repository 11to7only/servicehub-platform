# Non Functional Requirements

## Scalability

The system should support increasing number of users
and service providers without performance degradation.

## Availability

The system should remain operational even if
individual services fail.

## Reliability

Failures should be handled using retry mechanisms
and circuit breakers.

## Observability

The system should support monitoring and logging
to help detect and resolve incidents quickly.

## Security

All APIs should be protected using authentication
and authorization mechanisms.

## Performance

API responses should typically be returned
within 200 milliseconds.
